## Definition of Done for Every Product

A product is not complete until it has:

- internal use or a sanitized proof-of-work deployment;
- clear buyer and problem definition;
- documented scope;
- working files in mobile and desktop formats;
- sample screenshots using non-sensitive data;
- pricing tiers;
- setup and support boundaries;
- data-handling notice;
- landing-page copy;
- demo script;
- case study;
- pipeline record;
- approval gate before publication or customer delivery.

## Proof-of-Work Case Study Standard

Each case study must include:

- prior workflow;
- operational problem;
- implemented workflow;
- tools used;
- elapsed setup time;
- measured or observed result;
- limitations;
- data security controls;
- reusable assets created;
- next improvement.

Do not invent metrics. Record only measured or directly observed results.

- Configuration: $249-$499

- Advanced implementation: $1,500-$5,000+

### Recurring support

- Basic monitoring: $99-$199/month

- Retainer/custom operations support: separately scoped

Prices are starting ranges and must be approved before public release.

## Landing Page Requirements

Every page must include:

- plain-language problem statement;
- target buyer;
- specific deliverables;
- screenshots or proof-of-work evidence;
- pricing or request-a-quote path;
- clear CTA;
- FAQ;
- disclosure for any compensated recommendation;
- no unsupported savings, revenue, legal, compliance, or protection claims.

## Revenue Conversion Flow

  -> discovery call

  -> recurring support
