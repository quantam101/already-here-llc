## Quality Gate

Run the full quality gate (6 checks):
```bash
npm run qa:gate
```

This runs in order: lint, typecheck, build, test, seo-route-audit, content-guard. All must pass.

## Content Guard

Run the content guard standalone:
```bash
npm run qa:content
```

Expected output: `"status": "pass", "findings": 0`

The content guard (`scripts/a-plus-content-guard.mjs`) scans for forbidden patterns:
- SDVOSB-certified / SDVOSB Certified (should be "SDVOSB-eligible")
- set-aside / sole-source procurement language
- Dead contact routes (e.g. /contact)
- Encoded/obfuscated strings

Meta files (scripts, agent definitions, docs) are excluded via `skipFiles` set. Directories like `.git`, `node_modules`, `.next`, `out`, `already-here-llc-v1.1` are excluded via `ignoreDirs`.

## Lint & Typecheck

```bash
npm run lint
npm run typecheck
```

Note: `next-env.d.ts` is excluded from ESLint (auto-generated by Next.js build).

## Browser Verification

Key pages to check on the production site (`alreadyherellc.com`):

1. **Capability Statement** (`/capability-statement`)
   - Tab title should include "SDVOSB-Eligible Field Execution"
   - "SDVOSB Eligible" badge visible
   - No "SDVOSB-certified", "set-aside", or "sole-source" language

2. **Blog posts** (`/blog/{slug}`)
   - Blog renderer uses plain text interpolation (`{post.content}` with `whitespace-pre-line`)
   - Markdown `##` headings render as literal `##` text (not formatted headings)
   - HTML tags would render as literal text — use markdown or plain text only
   - Verify no forbidden SDVOSB language
