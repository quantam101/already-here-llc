# Daily Command Runbook

This runbook defines how to execute the Daily Command Summary consistently.

## Objective

Produce a daily decision report that filters connected operating signals into ranked actions for Already Here LLC.

The report must prioritize:

1. Revenue today
2. Hard deadlines
3. Payment/cash-flow protection
4. Production/security health
5. Vendor/client relationship movement
6. Drone/UAS certification and paid-work pipeline
7. AI lead-capture proof-of-work

## Inputs

Use only approved connected sources and public-safe research where appropriate:

- Gmail
- GitHub
- Google Calendar when schedule context is needed
- vendor/platform notifications
- procurement notices
- billing/payment notices
- Vercel/GitHub deployment alerts
- approved public sources for external verification

## Execution sequence

### Step 1 — Collect hot signals

Run searches for:

- deadlines within 7 days
- dispatch/work-order activity
- billing/payment items
- GitHub/website failures
- procurement deadlines
- drone/V2D/FAA/Part 107 items
- AI lead-capture pilot movement

### Step 2 — Remove noise

Suppress:

- duplicates
- unchanged repeated alerts
- expired opportunities
- low-value promotions
- newsletters without action
- generic job alerts without pay/location/scope
- items below effective revenue threshold unless strategically useful

### Step 3 — Determine material change

An item is material when it changes one or more of these:

- money expected, owed, due, or at risk
- deadline or schedule
- work acceptance, rejection, counter, or scope
- production health
- security/credential exposure
- legal/compliance risk
- eligibility for funding/training
- buyer/vendor relationship status
- proof-of-work progress

### Step 4 — Grade items

Use this grade scale:

- A+: immediate money, hard deadline, major security/production risk, or high-probability funded opportunity
- A: same-day action with clear upside or risk reduction
- A-: important but not fully blocking
- B+: meaningful opportunity or risk requiring controlled follow-up
- B: useful but not urgent
- C: monitor only
- D: low-value, suspicious, or likely negative action
- F: harmful, scam, non-fit, or do-not-touch

### Step 5 — Produce top 3 ranked actions

Top 3 must be chosen by:

1. deadline pressure
2. revenue/funding probability
3. downside risk avoided
4. operational leverage
5. fit with Already Here LLC strategy

### Step 6 — Write each item in command format

Required item structure:

```text
Source:
Date:
Subject/title:
Material change:
Deadline:
Eligibility:
Cost/free status:
Apply/contact link if available:
Grade:
Risk flags:
Exact next action:
Approval gate:
```

### Step 7 — Identify blocked actions

Clearly mark anything requiring Stephen Franklin's approval before execution.

Approval is required for:

- sending emails
- deleting or archiving emails
- submitting forms
- enrolling or applying
- accepting/countering/declining jobs
- moving money or paying invoices
- changing credentials
- modifying production files
- merging/deploying production code
- submitting personal, veteran, tax, banking, or company documents

### Step 8 — Store report safely

Public repo may store only:

- workflow
- prompts
- schemas
- sanitized examples
- non-sensitive templates

Private reports must not be committed to a public repository.

## Done definition

A Daily Command run is complete when:

- top 3 actions are listed
- all included items are actionable or materially changed
- each item has risk flags and exact next action
- approval gates are clear
- duplicates/noise are suppressed
- private data is not exposed to public repositories
